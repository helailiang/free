# 需求与功能

## 已确认需求

- 覆盖 C2/H1 网络通信测试完整路线图，但第一阶段优先落地命令行、Pytest 和自动报告。
- ROS 驱动测试暂不纳入本轮实现。
- H1/H2 命名统一按 H1 处理，协议依据 H1E0-02A 产品说明书第 4 章和表 4-2。
- C2 第一版复用现有 C200 压力测试脚本中的 2111/TCP、连续取数、停止取数和配置查询命令。

## 第一阶段功能

- 读取单台设备 JSON 配置。
- 执行 TCP 连接和 H1 登录。
- 执行基础协议查询。
- 启动连续取数并统计帧数、圈数、缺包率、解析错误。
- 执行应用层主动重连耗时验证。
- 生成 JSON、CSV、HTML 报告。
- 支持长稳 runner，并允许记录人工网络损伤事件文件路径。

## 暂缓功能

- GUI 封装。
- ROS2 topic 验证。
- 网络损伤仪自动控制。
- 高风险协议 fuzz、广播风暴和固件升级压力循环。
